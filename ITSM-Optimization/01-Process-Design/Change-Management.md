## Objective
Ensure all IT changes are assessed, approved, implemented, and reviewed in a controlled manner to minimize risk to municipal services.

## Scope
- Standard Changes (Pre-approved, low risk)
- Normal Changes (CAB approval required)
- Emergency Changes (Expedited approval)

## Change Lifecycle
1. Request for Change (RFC)
2. Impact & Risk Assessment
3. Technical Review
4. Change Advisory Board (CAB) Approval
5. Implementation
6. Post Implementation Review (PIR)
7. Closure

## Risk Assessment Factors
- Business Impact
- Number of users affected
- Security impact
- Rollback complexity
- Compliance risk

## Change Advisory Board (CAB)
Members:
- IT Operations
- Security
- Network
- Application Owners
- Business Representatives

## Emergency Change Process
- CIO / IT Manager approval
- Immediate implementation
- Mandatory PIR within 24 hours

## KPIs
- Change Success Rate
- Emergency Change Percentage
- Failed Change Rate
- Unauthorized Change Count
