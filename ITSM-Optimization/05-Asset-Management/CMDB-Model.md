# Configuration Management Database (CMDB) Model

## Objective
Maintain an accurate inventory of all IT assets and their relationships to improve change, incident, and problem management.

## CI (Configuration Item) Types
- Servers
- Network Devices
- Applications
- Cloud Services
- Endpoints (Laptop, Desktop, Mobile)
- Printers & Peripherals

## Relationships
- Server hosts Application
- Application used by Department
- Application supported by Vendor
- Endpoint assigned to User

## Governance
- Update CMDB during onboarding/offboarding
- Reconcile CMDB quarterly
- Link incidents, problems, and changes to relevant CIs
